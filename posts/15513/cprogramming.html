  • You can also make your own dynamic libraries in C and more!

Always explicitly cast pointer to other types and move to different levels of bytes (char*)A + i) to avoid the worry of moving to other locations, the pointer automatically takes care of the size of the type so that you do not have to worry about it

Only malloc when you have a persistent variable size data structure, or you have concurrent accesses (all threads can access stuff on the heap)

use valgrind --leak-check=full ./a.out <args> to check for memory leaks

in strcmp ensure that the src and dst do not overlap, also ensure that the strings are null terminated and ensure that the string actually contains n bytes.

Important functions can be

memset (void *ptr, int val, size_t n), set the locations to val for these n bytes. commonly used to to initialize a value to all zero bytes (usually for character arrays), since that is a byte level access

similarly we use memcpy(dst, src, bytes). They should not overlap, if they do I think memmove is the right way to go.
