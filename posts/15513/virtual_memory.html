The virtual memory

  • the page fault is an interrupt that the kernel handles once some data that you do not have is in the memory.
  • iret is used to resume your program when you are coming back from the interrupt process
  • the MMUis what maps your programs addresses to the virtual memory
  • Thrashing is when too many processes are using the disk bandwidth
  • Each process has its own virtual address space. It can view memory as a simple linear array.
  • sometiemes different Virtual addres spaces can map to the same loctions in the phsycial memory (a form of sharing) of read only code. read only code can go to the same place.

The Page table is a per process thing that is in the dram.

The page table contains the bits for each process - CR3 register that lives on the CPU, that holds the PTE value( points to the page table in SRAM cache/ DRAM), MMU uses that to go and obtain find the page table entry and copute the physical address needed, then the dram sends the data to the cpu.

the TLB is to prevent the repeated memory access The TLB is a cache for the page table entries.
