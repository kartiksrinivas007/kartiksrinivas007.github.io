Databases and SQL Notes

Important operations include - aggregate functions that average on a set of tuples that could be obtained after a reduction (using WHERE), a join and then grouping by and a next reduction using having.

10 tuples, or a small number of tuples that achieve something, (with ties) etc are all forms of output control, order by also is output control

You can select specific things according to the column number or you can also pass in string functions like SELECT UPPER(name) to modify the result of what you want to obtain. But this is done in the end clearly – for like a final change. Understanding what tuples that actually make up the answer seems like the core idea. Also remember that ‘%’ matches every string and ’_’ matches a single character.

You can use window functions to compte ranks over partitions in the table, something like a group by, but oyu can use this to insert new columns with ranks as well.

You can nest queries not only inside the from clause but also with conditions inside the WHERE clause. You can use statements like EXISTS, IN etc to check whether the given tuple satisfies the necessary conditions

Use lateral joins for references, or use the WITH clause for local temporary tables, they can also bind to the column names before hand. Use lateral joins when you have two attributes that need to be averaged over the same table but grouped over different attributes.
